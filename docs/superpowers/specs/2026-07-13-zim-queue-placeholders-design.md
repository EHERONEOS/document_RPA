# ZIM 队列占位入口设计

## 目标

为 ZIM 船司增加 `FL_ZIM_SI` 与 `FL_ZIM_VGM` 两个可消费队列，以及与现有船司一致的目录和路由结构。当前不实现任何 ZIM 网页自动化填单逻辑。

## 架构

新增 `app/Spider/ZIM/`：

- `router.py` 将完整队列名映射到客户入口。
- `FL_ZIM.py` 提供 `fl_zim_si` 和 `fl_zim_vgm` 两个入口。
- 每个入口显式抛出未实现错误，避免任务被标记为成功。

任务总分发器根据队列名第二段识别 `ZIM`，加载其路由器。环境变量示例将两个 ZIM 队列加入 `RPA_QUEUES`，使 Worker 可以启动相应消费者。

## 数据流与错误处理

`FL_ZIM_SI` 或 `FL_ZIM_VGM` 的消息经现有消费者构建任务上下文，再由总分发器转发至 ZIM 路由器和相应客户入口。入口在具体自动化逻辑补充前，以带队列名的明确异常终止；现有失败结果处理继续负责回传该失败。

未知的 ZIM 队列保持现有的路由未找到错误。WHL 及其他船司路由不改变。

## 验证

新增路由测试，覆盖：

- `FL_ZIM_SI` 分发至 SI 占位入口并产生预期未实现错误。
- `FL_ZIM_VGM` 分发至 VGM 占位入口并产生预期未实现错误。
- 现有 WHL 路由仍可解析。
