设计一个rpa自动化填单系统用于船司的自动化填单
# 虚拟环境通过uv搭建 python版本号定为3.12.12 MQ框架为funboost 自动化框架采用DrissionPage

# RabbitMQ框架采用funboost 开发 
 # 队列名格式 客户名_船司名_业务名的形式 若FL_WHL_SI
 # 队列信息通过nacos读取 第一段开发先走默认本地测试 连接地址为192.168.60.106:5672  账号密码都为guest

# 队列的消息模板为 ../mssage_list/msg_demo.json

# 队列接收到信息要执行以下流程  通知消息字段名为task
 ## 每个队列会建一个类方法 会配置一些参数 如文件下载地址、是否无痕模式启动等
 1. 在通知信息中读取关键参数 若消息id(task.rpaMessageId) 登录账号(task.websiteInfo.websiteAccount)密码(task.websiteInfo.websitePassword)等
 2. 深拷贝一份需要填入的数据task.content (后续没填一个数据会删除对应的字段；完成后检查深拷贝后的数据是否为空来判断是否有漏填的数据)
 3. 发送一个消息接口通知后台在处理这条队列消息 会传入rpaMessageId和队列名
 4. 开始启动DrissionPage浏览器获取一个page页面 中间会进行一些设置
   4.1 设置浏览器启动参数 如'--start-maximized','--ignore-certificate-errors-spki-list',(后续补充)
   4.2 配置Chrome prefs 如download.default_directory，download.prompt_for_downloa(后续补充)
   4.3 配置独立的用户目录 文件夹名为 f'{用户名}_{船司名}' 如 DEMO_WHL (便于相同账号可以复用缓存)
   4.4 根据配置参数(每一个队列类配置) 来判断是否开启无痕模式（默认不开启）设置不等页面完整加载等
   4.5 从端口9000到9200中按顺序找一个浏览器启动端口 若遇到端口占用则切一个空闲端口
   4.6 设置完浏览器后拿到一个新页面
 5. 然后执行登录操作 一个船司网站会封装登录方法
 6. 若登录成功开始执行业务操作
    6.1 由类参数判断是否开启录屏功能（线上运行环境默认是开启；本地开发默认是关闭）
    6.2 然后开始执行业务操作
    6.3 无论任务成功或失败 任务结束后 需要进行停止录屏操作
    6.4 任务无论成功或失败都要调用队列publish方法通知结果 如果任务失败 还要截图当前页面 将图片地址(会传入oss)和报错信息传入队列publish方法
    6.5 任务结束后关闭浏览器进程
  
# 开发建议
 1. 封装基本的dom操作步骤 如点击、填充、取text值、取input的val值等；在取值前需要判断元素是否存在 若元素不存在通过调用这个方法的参数判断是否返回报错信息（xx元素不存在）（默认为true）
 2. 一些公共参数要封装到基本类但是 继承的子类如果需要一些个性化的改变需要修改默写值 如浏览器的配置参数等
 3. 每个dom操作需要再本地打印日志 这个打印方法需要封装到公共方法里面 可以执行过程中随处调用
 4. 验证码识别方法（图鉴/云码等）也需要封装到独立文件
 5. 项目运行时通过环境变量来获取本地需要执行的队列集合，后续可以改成服务接口的获取
 6. 每一个船司会定义一个单独的目录大写 如万海的 会定义到Spider/WHL 文件里；
 7. 接收到消息会通过队列名中的船司名去对应船司目录读取router.py文件 这里是判断该队列会进入这个船司的哪个方法；而且同一个客户同一个船司会放到一个文件里
 8. 本地环境变量都统一放入.env文件中 每个方法加上中文备注说明
 9. 所有的方法最好都要加上简单的中文注释
 10. 所有还未定义方法先留好入口 后续加入（如oss上传、通知后台正在处理队列，队列结果回传等）
 11. 结合以上看一下有没有其他建议
